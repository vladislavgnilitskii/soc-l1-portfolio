```bash

# структура репозитория
mkdir -p 00-progress 01-process 02-windows-ad 03-siem 04-detection 05-network 06-writeups notes
```

# Неделя 2 — Windows: события, аудит, типы логона

> Главная неделя месяца. Ты здесь с нуля — темп плотный, но это ядро профессии.

### День 1. Windows-логи: как устроено

- [ ] Event Viewer, каналы: Security, System, Application
- [ ] Что такое Event ID, где включается аудит (Local Security Policy / GPO)
- [ ] Настроить политики аудита на клиенте и сервере (Logon, Account Management, Process Creation)

### День 2. Ключевые Event ID входа — наизусть

- [ ] **4624** — успешный вход
- [ ] **4625** — НЕудачный вход (не выход! выход = 4634/4647)
- [ ] **4672** — назначены спец-привилегии (вход привилегированной учётки)
- [ ] **4648** — вход с явными учётными данными (runas)

### День 3. Типы логона (Logon Type) — это спрашивают всегда

- [ ] **2** — интерактивный (за машиной)
- [ ] **3** — сетевой (SMB-шара, удалённый доступ) ← самый частый в алертах
- [ ] **5** — служба
- [ ] **10** — RemoteInteractive = **RDP**
- [ ] **11** — по кешированным учётным данным
- [ ] Понять разницу Type 3 vs 10 на примере (полез в шару vs сел за рабочий стол)

- **Не тащить Go/K8s/DevOps в резюме SOC.** На L1 не котируется. Максимум одна строка.

- **Не пытаться быть L2 сразу.** Правильные инстинкты L1 (эскалация вместо самодеятельности) — то, что реально спрашивают.
- **Не генерировать конфиги вслепую.** Если Claude Code/Gemini сделали конфиг — обязан объяснить каждую строку. Иначе повторится «иллюзия компетентности».
